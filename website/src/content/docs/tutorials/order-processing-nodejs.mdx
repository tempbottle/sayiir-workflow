Learn how to build a production-ready order processing pipeline with Sayiir and TypeScript. We'll start with a simple sequential flow and progressively add durability, retries, and parallelism.

## What We're Building

An order processing pipeline that:
- Validates incoming orders
- Charges payment (with automatic retries)
- Checks inventory
- Sends confirmation emails

By the end, you'll have a durable workflow that survives crashes and scales to handle thousands of orders.

## Project Setup

Create a new project and install Sayiir:

```bash
mkdir order-processor && cd order-processor
pnpm init
pnpm add sayiir
```

If you're using TypeScript (recommended):

```bash
pnpm add -D typescript @types/node
npx tsc --init
```

## Define Tasks

Tasks are the building blocks of workflows. Let's define our order processing tasks with proper error handling and retry policies:

```typescript
import { task, RetryPolicy } from "sayiir";

interface Order {
  orderId: number;
  customerEmail: string;
  amount: number;
}

interface ValidatedOrder extends Order {
  validated: true;
}

interface ChargedOrder extends ValidatedOrder {
  paymentId: string;
  charged: true;
}

const validateOrder = task("validate-order", (order: Order): ValidatedOrder => {
  if (order.amount <= 0) {
    throw new Error("Invalid amount: must be positive");
  }
  if (!order.customerEmail) {
    throw new Error("Customer email is required");
  }
  return { ...order, validated: true };
});

const chargePayment = task(
  "charge-payment",
  (order: ValidatedOrder): ChargedOrder => {
    // In production: integrate with Stripe, Square, etc.
    // const paymentIntent = await stripe.paymentIntents.create({
    //   amount: Math.round(order.amount * 100),
    //   currency: "usd",
    // });

    return { ...order, paymentId: "pay_123abc", charged: true };
  },
  {
    timeout: "30s",
    retry: {
      maxAttempts: 3,
      initialDelay: "1s",
      backoffMultiplier: 2.0,
    },
  }
);

const sendConfirmation = task(
  "send-confirmation",
  (order: ChargedOrder): string => {
    // In production: integrate with SendGrid, Postmark, etc.
    return `Order ${order.orderId} confirmed (payment: ${order.paymentId})`;
  }
);
```

Notice the task options configure:
- **timeout**: Fails if `chargePayment` takes longer than 30 seconds
- **retries**: Automatically retries failed payments up to 3 times with exponential backoff (1s, 2s, 4s)

## Build a Simple Workflow

Compose these tasks into a sequential workflow:

```typescript
import { flow, runWorkflow } from "sayiir";

const workflow = flow<Order>("order-processing")
  .then(validateOrder)
  .then(chargePayment)
  .then(sendConfirmation)
  .build();

const orderData: Order = {
  orderId: 1,
  customerEmail: "alice@example.com",
  amount: 99.99,
};

const result = await runWorkflow(workflow, orderData);
console.log(result); // "Order 1 confirmed (payment: pay_123abc)"
```

This works great for simple cases, but what happens if your process crashes during payment? The order might be charged twice on restart.

## Make It Durable

Switch to `runDurableWorkflow` to get automatic checkpointing and crash recovery:

```typescript
import { runDurableWorkflow, InMemoryBackend } from "sayiir";

const backend = new InMemoryBackend();

const status = runDurableWorkflow(workflow, "order-1", orderData, backend);

console.log(status.status); // "completed"
if (status.status === "completed") {
  console.log(status.output); // "Order 1 confirmed (payment: pay_123abc)"
}
```

Now if your process crashes:
- Tasks are checkpointed after each step
- On restart, the workflow resumes from the last checkpoint
- Already-completed tasks are never re-executed

**Testing durability**: Try running this twice with the same workflow ID. The second run will immediately return the cached result without re-executing tasks.

## Add Parallel Branches

Real order processing needs to do multiple things at once. Let's check inventory while charging payment:

```typescript
import { branch } from "sayiir";

interface InventoryResult extends ValidatedOrder {
  inStock: true;
}

const checkInventory = task(
  "check-inventory",
  (order: ValidatedOrder): InventoryResult => {
    // In production: query inventory database
    return { ...order, inStock: true };
  }
);

const finalize = task(
  "finalize",
  ([payment, inventory]: [ChargedOrder, InventoryResult]): string => {
    return `Order complete: paid=${payment.charged}, stock=${inventory.inStock}`;
  }
);

const workflow = flow<Order>("order-v2")
  .then(validateOrder)
  .fork([
    branch("payment", chargePayment),
    branch("inventory", checkInventory),
  ])
  .join("finalize", finalize)
  .build();

const status = runDurableWorkflow(workflow, "order-2", orderData, backend);

if (status.status === "completed") {
  console.log(status.output); // "Order complete: paid=true, stock=true"
}
```

The `fork()`/`join()` pattern:
1. Executes `chargePayment` and `checkInventory` in parallel
2. Waits for both to complete
3. Passes results as a tuple to `finalize` in declaration order

## Switch to Postgres

For production, use `PostgresBackend` for true durability across server restarts:

```typescript
import { PostgresBackend } from "sayiir";

// Connects and runs migrations automatically
const backend = PostgresBackend.connect(process.env.DATABASE_URL!);

const status = runDurableWorkflow(workflow, "order-3", orderData, backend);
```

Now your workflow state survives:
- Process crashes
- Server restarts
- Deployments


## Full Code Listing

Here's the complete working example:

```typescript
import {
  task,
  flow,
  branch,
  runDurableWorkflow,
  PostgresBackend,
} from "sayiir";

// Types
interface Order {
  orderId: number;
  customerEmail: string;
  productId: string;
  amount: number;
}

interface ValidatedOrder extends Order {
  validated: true;
}

interface ChargedOrder extends ValidatedOrder {
  paymentId: string;
  charged: true;
}

interface InventoryResult extends ValidatedOrder {
  inStock: true;
}

// Task definitions
const validateOrder = task("validate-order", (order: Order): ValidatedOrder => {
  if (order.amount <= 0) throw new Error("Invalid amount: must be positive");
  if (!order.customerEmail) throw new Error("Customer email is required");
  return { ...order, validated: true };
});

const chargePayment = task(
  "charge-payment",
  (order: ValidatedOrder): ChargedOrder => {
    // Integrate with payment provider
    return { ...order, paymentId: "pay_123abc", charged: true };
  },
  { timeout: "30s", retry: { maxAttempts: 3, initialDelay: "1s", backoffMultiplier: 2.0 } }
);

const checkInventory = task(
  "check-inventory",
  (order: ValidatedOrder): InventoryResult => {
    // Query inventory database
    return { ...order, inStock: true };
  }
);

const finalize = task(
  "finalize",
  ([payment, inventory]: [ChargedOrder, InventoryResult]): string => {
    return `Order complete: paid=${payment.charged}, stock=${inventory.inStock}`;
  }
);

// Workflow definition
const workflow = flow<Order>("order-processing")
  .then(validateOrder)
  .fork([
    branch("payment", chargePayment),
    branch("inventory", checkInventory),
  ])
  .join("finalize", finalize)
  .build();

// Execution
const backend = PostgresBackend.connect(process.env.DATABASE_URL!);

const order: Order = {
  orderId: 1001,
  customerEmail: "alice@example.com",
  productId: "widget-42",
  amount: 99.99,
};

const status = runDurableWorkflow(
  workflow,
  `order-${order.orderId}`,
  order,
  backend
);

if (status.status === "completed") {
  console.log(status.output);
}
```
